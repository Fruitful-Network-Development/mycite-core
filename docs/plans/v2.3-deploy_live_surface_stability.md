# V2.3 Deploy And Live Surface Stability

Authority: [v2-authority_stack.md](v2-authority_stack.md)

This follow-on plan is intentionally separate from the repo-and-contract datum
alignment work.

## Problem statement

Repo head and deployed V2 portal behavior have drifted apart:

- the repo contains the current trusted-tenant routes and surfaces
- the running FND Python host still behaves like an older admin-shell-only build
- live static JS already contains newer trusted-tenant render kinds

That mixed static/Python deploy state needs its own remediation pass.

## Scope

- redeploy both V2 portal services to current repo head
- unify build-id wiring across FND and TFF
- verify `/portal/home`, `/portal/status`, `/portal/activity`, and
  `/portal/profile-basics` are live on the deployed hosts
- add black-box smoke checks that compare:
  - repo route catalog
  - deployed static bundle
  - running host behavior

## Explicit non-goals

- no new datum semantics
- no new shell-region contracts
- no new portal slices
