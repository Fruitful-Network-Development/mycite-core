# Module Inventory

This inventory classifies the current `mycite-core` engineering surface after
the April 1, 2026 modularization pass. The goal is to keep the live runtime
stable while moving the durable ownership model toward capability modules
instead of portal-history paths.

| Area | Current path(s) | Status | Concern it owns | Concern leakage | Target module | Action needed |
| --- | --- | --- | --- | --- | --- | --- |
| Runtime entrypoints | `runtime/app.py`, `runtime/bin/run_portal.sh`, `portals/runtime/*` | Canonicalized with compatibility seam | Gunicorn startup and flavor loading | Legacy callers still import `portals/runtime/*` | `runtime/` | Keep wrappers until native service unit envs and docs stop referencing the old path |
| Runtime flavor loader | `portal_core/composition/runtime_loader.py`, `portals/runtime/app.py` | Correctly placed | Flavor module discovery and import reset | None beyond wrapper | `portal_core/composition` | Keep loader stable; do not add flavor behavior here |
| Instance context and state-root derivation | `portal_core/composition/instance_context.py`, `portal_core/shared/state_roots.py`, `portals/_shared/portal/application/runtime/instance_context.py` | Canonicalized with compatibility seam | Instance identity, state root, public/private/data path derivation | Flavor apps still build too much directly after context creation | `portal_core/composition`, `portal_core/shared` | Continue moving flavor bootstrap code to consume context instead of rebuilding paths |
| Shared runtime paths | `portal_core/shared/runtime_paths.py`, `portals/_shared/portal/runtime_paths.py` | Canonicalized with compatibility seam | Network, contract, utility-tools, request-log, vault, keypass paths | Some flavor services still compute adjacent paths manually | `portal_core/shared` | Move remaining direct path math behind helper functions |

Immediate result:

- the durable cross-cutting seams now live under `runtime/`, `portal_core/`,
  `tools/`, and `instances/`;
- the old `portals/_shared/...` locations still work, but they no longer need to
  remain the engineering source of truth for those concerns;
- the largest remaining mixed-concern file is
  `portals/_shared/portal/application/service_tools.py`;
- the largest remaining runtime-history seam is the pair of flavor app modules.
