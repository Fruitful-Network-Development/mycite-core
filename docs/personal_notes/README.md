# Personal Notes

Personal notes are preserved source material, not silent canon.

Personal notes were reset to match the one-shell portal model. Any notes that
encoded the retired split-shell assumptions were removed.

Archived MOS source-only notes now live under `docs/personal_notes/archive/MOS/`
so the active note path stays limited to material that still supports current
repo work.

## Promotion Targets

When note content becomes stable enough to reproduce for others, promote it into
the smallest correct surface:

- `docs/wiki/` for orientation, assignment, and explanatory maps
- `docs/contracts/` for normative and cross-package/cross-tool requirements
- `docs/plans/` for active work, backlog, and sequencing
- `docs/audits/` or `docs/audits/reports/` for evidence, validation, and review

Keep the original note unless it is purely redundant after promotion. The note
is allowed to remain as preserved source memory, but it should point operators
toward the canonical promoted surface when one exists.
