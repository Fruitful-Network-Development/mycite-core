How this works

The portal UI calls /portal/api/admin/control/.... NGINX proxies those calls to the localhost-only host control API, which applies narrow docker compose actions.

If you are viewing the portal directly on 127.0.0.1:5101 (no NGINX), these controls return auth/path errors.

Portal Instance Modes

Use this section to keep portal instances fully under /srv/compose/portals: set each instance to off or live, and assign auth_mode.

Mode semantics: off stops container, live keeps the instance running for routed access.
